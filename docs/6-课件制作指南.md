## 📐 教室大屏幕显示规范

### 1. 标准分辨率配置

#### 主要配置（推荐）

- **标准分辨率**：1920×1080像素（16:9）
- **适用设备**：现代教室大屏幕电视、高清投影仪
- **观看距离**：3-8米
- **使用场景**：95%的教学环境

#### 备选配置（兼容性）

- **备选分辨率**：1024×768像素（4:3）
- **适用设备**：老式投影仪、部分传统设备
- **使用场景**：设备较老的教室环境

### 2. Reveal.js 配置方案

#### 标准配置（1920×1080）

```javascript
// 主要配置 - 适用于大部分教室
Reveal.initialize({
    width: 1920,
    height: 1080,
  
    // 固定缩放，确保字体一致性
    minScale: 1.0,
    maxScale: 1.0,
  
    // 基本设置
    center: true,
    controls: true,
    progress: true,
    touch: false
});
```

#### 备选配置（1024×768）

```javascript
// 备选配置 - 适用于老式设备
Reveal.initialize({
    width: 1024,
    height: 768,
  
    // 固定缩放
    minScale: 1.0,
    maxScale: 1.0,
  
    // 基本设置
    center: true,
    controls: true,
    progress: true,
    touch: false
});
```

### 3. 简化布局标准

#### 布局结构

- **标题区域**：顶部20%，使用大字体
- **主内容区域**：中央70%，图文并茂展示
- **导航区域**：底部10%，简洁的页码和控制

#### 标准分辨率样式（1920×1080）

### 4. 配置选择指南

#### 如何选择配置

1. **检查设备分辨率**：查看教室大屏幕的原生分辨率
2. **优先使用标准配置**：1920×1080适用于大部分现代设备
3. **兼容性考虑**：老式设备使用1024×768配置
4. **测试验证**：在实际设备上测试显示效果

### 3. Web字体使用指南

#### 跨平台字体配置标准

##### 统一字体配置原则

- **兼容性优先**：确保Windows和macOS系统都能正确显示
- **降级机制**：提供完整的字体回退方案
- **教学优化**：选择适合大屏幕显示的清晰字体

##### 推荐字体选择

- **中文字体**：'Microsoft YaHei', 'Heiti TC', 'SimHei', sans-serif
- **英文字体**：'Arial', 'Helvetica', 'Calibri', sans-serif
- **数字字体**：'Consolas', 'Monaco', 'Courier New', monospace

##### Web字体配置

```css
/* 引入Web字体作为补充 */
@import url('https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@300;400;700&display=swap');

/* 跨平台字体配置 */
.reveal {
    font-family: 'Noto Sans SC', 'Microsoft YaHei', 'Heiti TC', 'SimHei', sans-serif;
}

.reveal .slides section h1,
.reveal .slides section h2 {
    font-weight: 700;
}
```

#### 字体大小标准化

- **主标题**：3-4rem（确保远距离可见）
- **副标题**：2-2.5rem
- **正文内容**：1.5-2rem
- **图表标注**：1.2-1.5rem
- **页脚信息**：1-1.2rem

#### 动画效果时序检查

- [ ] 复杂动画提供"跳过"选项

## 🎯 设计原则

### 图文并茂核心要求

- **图像占比≥70%**：每页幻灯片以视觉元素为主
- **文字≤30%**：文字仅用于标题、标注和关键信息
- **视觉优先**：用图表、图像传达主要信息
- **简洁明了**：避免信息过载，突出重点

### Reveal.js响应式设计原则

- **移动优先**：优先考虑触控设备的用户体验
- **弹性布局**：使用CSS Grid和Flexbox实现自适应布局
- **渐进增强**：基础功能在所有设备上可用，高级功能逐步增强
- **性能优化**：图片懒加载，CSS/JS文件压缩

### 年龄适应性设计

- **字体大小**：标题≥3rem，正文≥1.5rem（相对单位）
- **颜色对比**：高对比度配色，考虑色盲学生
- **动画效果**：使用CSS3 transition，避免JavaScript重动画
- **交互设计**：支持键盘、鼠标、触控多种输入方式

### Web可访问性原则

- **语义化HTML**：使用正确的HTML标签结构
- **键盘导航**：支持Tab键和方向键导航
- **屏幕阅读器**：添加alt属性和aria标签
- **色彩无障碍**：不仅依赖颜色传达信息

## ⏰ 45分钟课时时间规划

### 标准课时结构（与课程详细内容一致）
- **情景导入（5分钟）**：短视频或图片引入，提出1–2个引导问题。
- **基础知识（15分钟）**：核心概念讲解与图文示例，强调关键词与要点。
- **互动实践（15分钟）**：基于图片/数据的交互观察或简化分析，含电子记录。
- **总结与讨论（10分钟）**：要点回顾、科学表达与延伸思考。

### 时间分配建议（对应幻灯片页数）
- **导入**：1–2页（封面/目标页/情景图片/视频说明）。
- **基础**：5–7页（概念介绍、对比示例、关键术语）。
- **互动**：4–5页（操作提示、交互截图/说明、问题卡片）。
- **总结**：2–3页（要点清单、思考题、课后任务）。

### 互动环节设计要点
- 每次操作目标明确、步骤不超过3步；
- 提供即时反馈（高亮、标注、切换）；
- 预置“电子记录卡片”，引导标准科学表述。

### 弹性调整方案（保45分钟总时长）
- **实践偏重型**：5/10/15/15（导入/基础/互动/总结）。
- **讲解偏重型**：5/20/10/10（导入/基础/互动/总结）。
- 调整时保持“互动≥10分钟、总结≥8分钟”。

> 一致性依据：详见 `docs/2-课程详细内容.md` 各课“教学活动”分配（5/15/15/10）。

### 幻灯片数量与时间对应标准

#### 基础原则

- **每张幻灯片平均停留时间**：2-3分钟
- **总幻灯片数量**：12-18页（不含封面和目录）
- **核心内容幻灯片**：8-12页
- **互动环节幻灯片**：2-4页

#### 不同类型幻灯片的时间分配

| 幻灯片类型           | 建议时间 | 使用场景 | 设计要点           |
| -------------------- | -------- | -------- | ------------------ |
| **封面页**     | 30–45秒  | 导入环节 | 简洁明了，吸引注意 |
| **目标页**     | 1–2分钟  | 导入环节 | 清晰列出学习目标   |
| **情景导入页** | 2–3分钟  | 导入环节 | 短视频/图片引入与提问 |
| **概念介绍页** | 2–2.5分钟| 基础知识 | 图文并茂，重点突出 |
| **互动操作页** | 3–4分钟    | 互动实践 | 操作简单，反馈及时 |
| **讨论思考页** | 3分钟     | 总结讨论 | 问题明确，层次清晰 |
| **总结页**     | 2–3分钟  | 总结讨论 | 要点突出，便于记忆 |

> 说明：以每课 12–18 页为基准，精确映射 45 分钟（5/15/15/10）。导入（封面+目标+情景导入）≈5分钟；基础（概念介绍页）≈15分钟；互动（互动操作页）≈15分钟；总结与讨论（讨论思考+总结）≈10分钟。若互动选4页，建议每页3.5–4分钟；若选5页，建议每页≈3分钟。

plt.rcParams['axes.unicode_minus'] = False    # 解决负号显示问题

```python
import matplotlib.pyplot as plt
plt.rcParams['font.family'] = 'Heiti TC'      # 黑体-繁（系统自带）
